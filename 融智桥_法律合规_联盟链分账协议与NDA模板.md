# 融智桥 · 法律合规与协议模板（联盟链 + 法币 / NDA）

> **文档性质**：法律专家 **Draft（草案）**，供内部评审与后续执业律师终审使用，**不构成法律意见**。
> **版本**：v0.1　**适用阶段**：DEMO → MVP 预研
> **合规基线**：中国法项下——「币链分离」监管（银发〔2026〕42 号）、个人信息保护法（PIPL）、电子签名法、反洗钱（AML/KYC）相关规定。
> **一句话定位**：本平台**不触碰公链代币 / 稳定币**；采用**许可联盟链 + 人民币法币 + 持牌资金托管**的「透明智能分账协议」模式。

---

## 一、适用范围与合规总纲

### 1.1 协议主体（四方）
| 角色 | 法律主体性质（草案建议） | 备注 |
|------|------------------------|------|
| 行业专家（35+） | 自然人 / 个体户 / 顾问服务主体 | 可提供企业顾问、教练、方案 |
| AI 技术人才 | 自然人 / 技术服务主体 | 负责 Agent / 方案工程化 |
| 企业客户（代表） | 企业法人 / 经办人 | 需求方、付款方 |
| 平台方 | 运营公司（融智桥） | 提供撮合、规则、仲裁、托管对接 |

### 1.2 监管红线（刚性禁止）
- ❌ **禁止**发行或流转任何虚拟货币 / 公链代币 / 稳定币作为结算手段；
- ❌ **禁止**平台自建资金池或无支付牌照开展资金清算；
- ❌ **禁止**以「W3 / 公链合约」包装实质为代币融资的话术；
- ✅ **允许**在**许可联盟链**上以智能合约表达分账**规则与记账**，资金以**人民币**经**持牌机构**清算。

> 依据：2026-02 八部委《关于规范区块链技术在金融服务中应用的指导意见》（银发〔2026〕42 号），明确「币链分离」——链上仅承载合约与存证，价值结算须由持牌金融机构以法定货币完成。

### 1.3 数据合规基线
- 名片字段（姓名/电话/微信/职务/介绍）属个人信息，须遵循 **PIPL**：告知—同意、目的限定、最小必要、可撤回；
- DEMO 阶段全部为**演示数据**，不处理真实个人敏感信息；正式产品上线前须完成个人信息影响评估（PIA）；
- 链上仅存**哈希 / 凭证 / 规则**，不链上明文存储个人数据。

## 二、联盟链 + 法币「透明智能分账协议」条款（草案）

> 对外统一话术：**「透明智能分账协议」**（不称「链上发币」「代币激励」）。

### 2.1 定义
- **里程碑（Milestone）**：专家、AI 人才、企业三方在 DEMO / 正式项目中约定的可验收交付节点。
- **分账规则（Split Rule）**：由三方在联盟链上以智能合约形式约定，明确各方应得比例与触发条件。
- **2-of-3 多签（Multi-sig）**：专家签章 + 企业签章 + 平台仲裁，三方中两方（含平台仲裁）确认即触发结算。
- **法币清算（Fiat Settlement）**：以人民币（CNY）经持牌托管/支付机构完成实际资金划转。

### 2.2 分账触发
1. 企业代表在平台确认里程碑达成（游戏化审阅 / 验收）；
2. 系统生成「验收凭证」哈希并上链存证；
3. 触发 2-of-3 多签流程（§2.3）；
4. 多签达成后，向持牌清算机构发送**法币分账指令**。

### 2.3 2-of-3 多签与平台仲裁
- **签章方**：专家（确认交付）、企业（确认验收）；
- **仲裁方**：平台（对规则解释、争议裁决、异常兜底）；
- **生效条件**：专家 + 企业双方签章，**或** 任一交付方签章 + 平台仲裁签章；
- **争议场景**：企业拒签且理由不成立 → 平台依证据仲裁支持专家/AI；企业合理拒签 → 触发返工，不触发分账；
- **留痕**：每次签章含时间戳、角色标识、上链交易哈希，可供司法取证。

```text
[专家签章] ─┐
            ├─ 2-of-3 → [持牌机构法币分账指令] → [人民币到账]
[企业签章] ─┤
            │
[平台仲裁] ─┘  （兜底 / 争议裁决）
```

### 2.4 法币清结算路径（持牌托管）
- 平台**不直接收付款**，资金由**银行存管 / 持牌第三方支付**托管；
- 联盟链智能合约仅输出「分账指令（谁、多少、依据）」，由托管机构执行实划；
- 资金流：企业付款 → 托管账户 → 按分账规则 → 专家 / AI 人才 / 平台（take rate）；
- 平台 take rate 与链上清算费分离计价（见 §2.5），避免「代币费率」歧义。

### 2.5 费率结构（两段式，透明公示）
| 费用项 | 口径 | 区间 | 收取方 |
|--------|------|------|--------|
| 平台 take rate | 项目金额抽成 | **10–20%** | 平台方 |
| 链上清算费 | 联盟链记账/存证 + 持牌清结算服务费 | **1–3%** | 平台代付 / 代收后转嫁 |

> 说明：两段式是为消除「平台抽成 10–20%」与「链上 1–3% 清算费」口径混淆（参考竞品 Catalant 20–30% 抽成、智汇联 1–3% 上链费）。**最终以平台公示价目表与单个协议约定为准**。

### 2.6 链上留痕与审计
- 上链内容：分账规则哈希、验收凭证哈希、签章记录、分账指令回执哈希；
- 不上链：个人身份信息、合同全文、商业秘密原文；
- 审计：平台保留可调取的链上凭证映射，配合监管与争议调查；
- 不可篡改：规则与签章一旦上链，单方无法篡改，降低赖账风险。

### 2.7 KYC / 反洗钱
- 企业须完成实名认证（营业执照 + 经办人）；专家 / AI 人才须实名（身份证 / 执业资质）；
- 单笔 / 累计交易超阈值触发反洗钱申报义务（按支付机构与银行要求）；
- 禁止为明显异常、涉诈、涉赌资金提供分账通道。

### 2.8 税务与发票
- 专家 / AI 人才取得报酬须依法申报个税（劳务报酬 / 经营所得）；
- 平台就 take rate 开具服务费发票；
- 系统须支持「分账明细 + 凭证」导出，便于各方报税与对账。

### 2.9 争议与仲裁
- 优先平台居中调解（依 §2.3 仲裁机制）；
- 调解不成，提交**平台运营地有管辖权法院**或约定**仲裁机构**；
- 链上签章与验收凭证可作为电子证据（依《电子签名法》可靠电子签名规则）。

### 2.10 协议终止
- DEMO 到期 / 未续期 → 自动失效，平台一键撤销参与方 API Key 与临时权限；
- 正式协议解除 → 已完成里程碑按规则结算，未完成部分终止，数据依规留存 / 删除。

## 三、保密协议（NDA）模板（DEMO 专用）

> 使用场景：专家、AI 人才、企业代表、平台方参与「生产排产与交期预警」等 DEMO 前在线签署。
> 形式：平台内在线签署，含**动态数字水印**（签署方钱包公钥 + 注册信息），到期撤销 API Key。

### 保密协议（NDA）· 草本

**甲方（平台/参与方）**：________________（企业名称 / 个人姓名）
**乙方（参与方）**：________________（企业名称 / 个人姓名）
**协议编号**：RZQ-NDA-________　**签署日期**：________

**鉴于**：双方就「________________ DEMO」（下称"本 DEMO"）开展协作，乙方将接触甲方及他方提供的业务逻辑、脱敏数据与方案草案，双方就保密义务约定如下：

**第 1 条 保密信息**
1.1 本协议所称保密信息，指在本 DEMO 中获悉的：业务流程逻辑、脱敏订单/排产数据、方案草案、规则设定、以及一方明确标示为保密的内容。
1.2 下列信息不属于保密信息：已公开（非因违约）、接收前已合法持有、或第三方合法提供的。

**第 2 条 保密义务**
2.1 接收方不得以任何形式截屏外传、转发、或逆向提取核心 AI 逻辑与底层方案。
2.2 接收方仅可在本 DEMO 目的范围内使用保密信息，不得用于自身经营或向第三方披露。
2.3 数字水印：本协议签署后即附加动态数字水印（含签署方钱包公钥与注册信息），用于泄露溯源；任何去除水印行为视为违约。

**第 3 条 DEMO 期限与权限撤销**
3.1 本 DEMO 设失效期，到期未续期自动失效。
3.2 失效或终止时，平台有权**一键撤销**乙方 API Key 与临时数据权限，乙方应停止使用相关接口与数据。

**第 4 条 违约责任**
4.1 任何一方违反本协议，应赔偿守约方因此遭受的直接损失，并承担合理的维权费用（含律师费、取证费）。
4.2 情节严重的，平台有权终止其参与资格并保留追究法律责任的权利。

**第 5 条 其他**
5.1 本协议适用**中华人民共和国法律**；争议提交平台运营地有管辖权法院。
5.2 本协议自双方在线签署（含电子签章/多签确认）之日起生效，至 DEMO 失效后 **__** 个月止（或另行约定）。
5.3 本 DEMO 全部数据均为**演示数据**，不构成真实交易或要约。

## 四、入驻与平台规则要点（草案，供下一步细化）

1. **名片字段**：姓名（昵称）/ 电话 / 微信 / 职务 / 个人介绍（重点）——须取得用户明示同意并标注用途；

4. **角色介入时间表**：各方角色在 DEMO→MVP→全功能各阶段介入节点，由对应专家后续明确。

## 六、合规红线速查（给产品/研发的提醒）

| 表述 | 合规 ✅ | 违规 ❌ |
|------|--------|--------|
| 价值结算 | 人民币法币 + 持牌托管 | 代币 / 稳定币 / 公链支付 |
| 链上内容 | 规则 / 凭证 / 哈希存证 | 明文个人数据 / 资金 |
| 对外话术 | 透明智能分账协议 | 链上发币 / 代币激励 |
| 资金流 | 银行/支付机构托管清分 | 平台资金池 |
| 签约 | 电子签章 + NDA + 多签留痕 | 无签署 / 无留痕 |
