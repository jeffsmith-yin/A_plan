## 四、产品概要设计要点

### 4.1 技术栈建议（初稿）

| 层次 | 建议方案 | 理由 |
|------|---------|------|
| 前端 | React + TypeScript + TailwindCSS | 生态成熟、招聘容易、支持多平台 |
| 后端 | Node.js (NestJS) 或 Go | 高并发支持、微服务友好 |
| 区块链 | Ethereum/Polygon (Layer 2) 或 联盟链 | 智能合约成熟度高、Gas费低 |
| 数据库 | PostgreSQL + Redis | 关系型+缓存经典组合 |
| 部署 | Docker + K8s（Web端优先） | 支持后续多平台扩展 |

### 4.2 保密协议模板要点

- 保密范围：DEMO展示的所有产品功能、商业模式、技术方案、演示数据
- 违约责任：明确赔偿条款
- DEMO有效期：约定起止日期
- 知识产权归属：平台方所有
- 签署方式：DEMO阶段勾选确认（非法律效力），正式版电子签

### 4.3 License模式设计

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| 按年收费 | 年付，到期续费 | 持续使用的中小微企业 |
| 永久 | 一次性买断 | 预算充足、希望锁定成本的企业 |
| 到期失效 | 设定截止日期自动失效 | DEMO版、短期项目 |
