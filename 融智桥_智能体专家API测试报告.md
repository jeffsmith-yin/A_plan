## 三、安全评估

### 3.1 已通过（不崩溃 / 正确拦截）
- **SQL 注入**（`'; DROP TABLE skills; --`）：返回 200，词法检索不触达任何数据库，无注入面；不崩溃。✅
- **XSS**（`<script>alert(1)</script>`）：载荷作为纯文本保留在 JSON 字段中，未被执行；前端页面用 `textContent` 渲染，无 DOM XSS。✅
- **畸形 JSON**：映射为 **400**（`readJson` 统一解析）。✅（修复前会返回 500 甚至挂起）
- **超大负载**（> 1 MB）：触发 **413** 体限，防 DoS。✅（修复前无上限）

### 3.2 标记缺口（MVP 阶段可接受，生产必须补齐）
| 项 | 现状 | 风险 | 生产建议 |
|----|------|------|----------|
| 鉴权 | 所有端点无 Bearer Token | 任意调用者可触发分析与结算 | 网关/中间件加 JWT，结算类端点强制鉴权 |
| 限流 | 无（120 并发请求全部放行，无 429） | 易被滥用/刷量 | 网关层按 IP/Token 限流（如 60 req/min） |
| 传输 | 仅 HTTP（演示） | 中间人篡改 | 正式环境强制 HTTPS/TLS |
| 购物车隔离 | 单进程内存、全局共享 | 多用户串号 | 按 session/user 隔离 + 持久化（接已交付 4 的 `uni.*` 存储或后端） |
| 审计日志 | 无请求/结算留痕 | 分账争议难追溯 | 结算走链上存证 + 服务端审计日志（接已交付 5 透明智能分账协议） |
| 输入 Schema | 仅校验 ID 存在性 | 字段类型/范围未约束 | 加 JSON Schema 校验 |

## 四、测试中发现并修复的缺陷（已交付 8/9 → 10 回馈）

| # | 严重度 | 缺陷 | 根因 | 修复 |
|---|--------|------|------|------|
| 1 | 🔴 高 | `POST /cart/add` 对**不存在的技能包 ID 返回 `ok:true`**，幽灵条目进入购物车，结算时被静默丢弃 | `addToCart` 未校验技能是否存在 | `cart.js` 改为先查知识库 `kb.skills`，不存在即 `return false` |
| 2 | 🔴 高 | **畸形 JSON 导致服务挂起**（客户端 8s 超时） | `readJson` 的 `responded` 标志在解析前被置真，`fail(400)` 提前 return 不发响应 | 仅在解析**成功**后置 `responded=true`；解析失败走 `fail(400)` 正常回 400 |
| 3 | 🟡 中 | 请求体**无大小上限**（5 MB 负载被全盘读入，DoS 风险） | 原始实现无体积约束 | `readJson` 加 1 MB 体限，超限回 **413** 并 `req.destroy()` |

> 上述修复已随本测试一并落地于 `src/cart.js`、`src/server.js`，并通过回归（20/20）。

## 五、问题、风险与优化建议

### 5.1 上线前必须（P0）—— 已补齐（代码落地，见 §九）

> 原三项 P0（鉴权+限流 / 购物车隔离持久化 / 结算审计）均已在本轮实现并通过 27/27 回归测试。以下为补充建议。

1. **鉴权 + 限流（已落地）**：`/cart/*`、`/audit` 强制 HMAC 令牌；全局固定窗口限流（默认 600/60s，按 token.sub 或 IP 隔离）。生产建议：替换为正式 IdP/公钥验签 + 刷新令牌 + 网关/Redis 分布式限流，密钥托管。
2. **购物车隔离与持久化（已落地）**：按 `sub` 隔离、落盘 `data/cart-store.json`。生产建议：改为后端用户库/会话服务，密钥加密存储。
3. **结算审计（已落地）**：每次 `checkout` 写审计日志 + 哈希链存证（`data/chain-ledger.jsonl`，篡改即断裂）。生产建议：审计入服务端持久库、存证上真实联盟链（TBaaS/蚂蚁链，接已交付 4 `ChainAdapter`、已交付 5）。

### 5.2 建议优化（P1–P2）
- **检索升级**：`rag.js` 词法匹配 → embedding 向量检索（语义召回更准，已交付 7 §3.3）。
- **生成升级**：规则拼接 → LLM + 行业 Skills 提示词，输出更自然可控。
- **多域扩展**：当前仅制造业，扩至零售/餐饮/外贸等域，每域一个智能体专家。
- **输入校验**：引入 JSON Schema 对 `text`/`id` 做类型与范围约束。
- **可观测性**：加请求日志、错误率、P95 延迟埋点，接已交付 4 数据看板。

### 5.3 已固化资产
- `test/api-test.js`：零依赖、可重复、可作为 CI 验收门禁（`npm test`，退出码 0=通过 / 1=失败 / 3=超时兜底）。
- 单次运行 < 1 秒、无外部依赖，易于接入流水线。

## 九、安全项 P0 实施说明（本轮补齐，27/27 通过）

### 9.1 鉴权（`src/auth.js` + `server.js`）
- HMAC(HS256) 签名令牌：`signToken` 含 `sub/role/iat/exp`；`verifyToken` 校验签名与过期；`login(apiKey)` 用演示允许列表签发（member / platform 两类 Key）。
- 受控端点：`/cart`、`/cart/add`、`/cart/checkout`、`/cart/clear` 强制 Bearer；`/audit` 额外要求 `role=platform`。
- 无 token / 伪造 / 过期 → 401；越权 → 403。
- **生产替换**：正式 IdP、公钥验签、刷新令牌、密钥托管（env `RZQ_JWT_SECRET`）。

### 9.2 限流（`src/ratelimit.js` + `server.js`）
- 全局固定窗口，按 `token.sub` 或 IP 隔离；默认 600/60s（env `RATE_MAX`/`RATE_WINDOW_MS`）；超限 → 429 + `Retry-After` + `X-RateLimit-*` 头。
- **生产替换**：网关 / Redis 分布式限流。

### 9.3 购物车隔离 + 持久化（`src/cart.js`）
- 购物车按 `sub`（鉴权主体）隔离，杜绝多用户串号；落盘 `data/cart-store.json`，重启可恢复。
- 保留 `'default'` 会话以兼容 CLI（`src/index.js` 无需改动）。
- **生产替换**：后端用户库 / 会话服务，密钥加密存储。

### 9.4 结算审计 + 哈希链存证（`src/audit.js` + `server.js`）
- 每次 `checkout` 写 `data/settlement-audit.jsonl`（含 sub / 金额 / 分成 / 时间）与 `data/chain-ledger.jsonl`（哈希链 `hash = H(prevHash | ts | sub | total)`，篡改即断裂，可由 `verifyLedger()` 校验）。
- `GET /audit`（platform）返回审计条目与链完整性，支撑「透明智能分账协议」分账争议追溯（接已交付 5）。
- **生产替换**：审计入服务端持久库、存证上真实联盟链（TBaaS / 蚂蚁链，接已交付 4 `ChainAdapter`）。

### 9.5 测试覆盖（对应 §一）
- 鉴权 4 项（无 token / 伪造 / 过期 → 401、有效 → 200）、角色越权 1 项（403）、限流 1 项（→ 429）；
- 隔离 2 项（用户 B 不可见 A、落盘含该用户）、审计 1 项（写审计 + 链完整）。
