## 1. 产品概述

### 1.1 背景与立项依据
市场调研已确认（见《融智桥市场调研与竞品分析》）：
- **供给侧**：中国 **35 岁以上、经验丰富但再就业困难**的行业专家（≠ 退休专家）海量闲置，有强行业 Know-How 但缺乏 AI 工具能力；
- **需求侧**：中小微企业超 **5200 万户**、90% 有数字化转型意愿，但 **70%+ 因缺人才/成本高/方案不接地气而失败**，核心业务 AI 深度应用率仅 **5.6%**、一年内 AI 项目停滞率 **68%**；
- **连接侧**：AI 人才缺口超 **500 万**，"有技术缺场景"，需行业专家给场景；
- **空白**：现有竞品（在行/猪八戒/GLG/Catalant/知微行易/链盟等）**均未同时覆盖"四方角色 + 游戏化 + NDA + 合规自动分账"**，且中小企业与 35+ 专家是结构性空白。

### 1.2 产品愿景与定位（一句话）
> **融智桥 = 把"不懂 AI 但有真本事的 35+ 专家"与"懂 AI 但缺场景的人才"粘合起来，用游戏化 DEMO + 保密协议 + 联盟链合规分账，向中小微企业交付"专家诊断 + AI 实现 + 企业验收"的可信解决方案平台。**

### 1.3 本次 MVP 范围
| 在范围内（MVP） | 不在范围内（后续阶段） |
|------------------|------------------------|
| 四方入驻 + 名片、实时交流、NDA、DEMO 闯关、联盟链+法币分账（里程碑触发）、积分推荐、数据看板、三种收费模式（LICENSE/永久/失效） | 多语言版本（先仅简体中文）、App/PC 原生（先微信小程序 + H5）、自建应用市场、复杂技能包交易市场、AI 自动生成方案 |

### 1.4 成功标准（MVP 验收量化目标）
- 招募 **30 名**供应链/快消/传统制造等领域 35+ 专家种子 + **20 名** AI 人才（据链盟 STEP2 / DeepSeek 验证目标）；
- 对 **20–30 家**目标企业完成 DEMO 内部测试；
- 核心验证三件事：① 专家愿为封装经验配合学一点 AI；② 企业愿为"人工+AI"预付小额验证费；③ 联盟链自动分账真打消回款顾虑。

## 2. 用户角色与画像

### 2.1 四方角色
| 角色 | 核心诉求 | 痛点 | 平台价值 |
|------|----------|------|----------|
| **35+ 行业专家** | 把经验变现、获得持续项目/被动收入 | 不懂 AI、经验难低成本产品化、怕品牌被挪用 | 专家端极简界面 + AI 人才实现 + 技能包沉淀 |
| **AI 技术人才** | 技术对口变现 | 缺行业场景、接单分散、信任成本高 | 专家给场景 + 平台派单 + 自动分账 |
| **中小微企业代表** | 看得懂、能试用、见效快、低成本 | 不会用 AI、付不起大咨询费、怕泄密 | 游戏化 DEMO + NDA + 按效果付费 |
| **平台运营方** | 透明自动化低信任摩擦的协作与分账 | 需保护各方 IP 与商密、三方冷启动 | 数据看板 + 规则治理 + 推荐裂变 |

### 2.2 入驻名片字段（暂定，须开发）
按项目规范，入驻采用"名片"方式，字段如下（**个人介绍为重点**）：
- 姓名（或昵称）
- 电话
- 微信
- 职务
- **个人介绍（重点）**：行业年限、擅长领域、代表案例、可交付的 AI 方案方向

> 设计原则（据 DeepSeek 版风险）：**专家端必须"极傻瓜化"**，降低 35+ 专家学习畏难；名片个人介绍应支持"专家口述 → AI 人才协助生成结构化卡片"，避免专家自写长文。

## 3. 核心业务流程（三方协作闭环）

```
① 企业提出痛点 ──→ ② 专家诊断（定义标准/规则/场景）
        │                      │
        │                      ▼
  ⑥ 联盟链分账 ◄── ⑤ 企业验收（游戏化审阅/打回/批准）
        ▲                      ▲
        │                      │
        └── ④ AI 人才实现（配 Agent/技能包，演示数据）
```

1. 企业代表在平台描述痛点，平台撮合/推荐 35+ 专家；
2. 专家诊断业务、定义交付标准与敏感规则（无需写代码）；
3. AI 人才据专家规则构建轻量方案/技能包（MVP 阶段可由内部 AI 人才或"模拟专家"保底，见 §9）；
4. 企业代表通过游戏化界面实时审阅、反馈、打回、批准；
5. 每完成一个约定里程碑，触发**联盟链 + 法币自动分账**（阶段解锁）；
6. 平台抽成（take rate 参考 10–20%）+ 链上清算费（1–3%），详见 §4.5。

## 4. 功能需求（按模块，含优先级）

优先级说明：**P0** = MVP 必须；**P1** = MVP 宜有；**P2** = 后续阶段。

### 4.1 入驻与身份管理（名片） — P0
- **FR-4.1.1** 四方角色（专家/AI 人才/企业代表/平台方）均可入驻，填写名片字段（§2.2）。
- **FR-4.1.2** 实名鉴权：微信活体检测 / 实名（兼顾合规与积分防刷，见 §4.6）。
- **FR-4.1.3** 角色可切换视角（同一用户可兼具专家与 AI 人才身份，但交易时明确当前角色）。
- **FR-4.1.4** 专家名片"个人介绍"支持 AI 辅助生成草稿，专家确认后发布。

### 4.2 实时交流与进度反馈（游戏化） — P0
- **FR-4.2.1** 三方（专家/AI 人才/企业代表）可在项目内实时 IM 沟通，反馈进展、提出问题。
- **FR-4.2.2** 提供"进度时间线 + 修改留痕"，所有反馈与版本可追溯。
- **FR-4.2.3** 基础互动以**游戏形式**呈现（如"寻宝式"业务配置导览、点选/拖拽审阅），降低企业代表与专家的使用门槛（据链盟报告）。

### 4.3 保密协议（NDA） — P0
- **FR-4.3.1** 平台与参与 DEMO 人员（专家/AI 人才/企业代表）在线签署保密协议（模拟/正式）。
- **FR-4.3.2** 动态数字水印：含签署方钱包公钥 + 注册信息，防截屏泄露（据链盟技术评估）。
- **FR-4.3.3** 核心 AI 逻辑保留在平台云端，客户端仅开放前端 UI 与 API 调用；**DEMO 到期一键撤销 API Key**（据链盟报告）。

### 4.4 DEMO 闯关与游戏化交互 — P0
- **FR-4.4.1** DEMO 与最终产品形态一致，支持在线交流，先提供最基本互动功能（游戏形式）。
- **FR-4.4.2** **功能开放度 ≤ 50%，核心功能 ≤ 20%**（据项目规范 + 链盟/DeepSeek 报告）。
- **FR-4.4.3** 全部数据为**演示用数据**，界面显著标注"演示"，非真实数据。
- **FR-4.4.4** DEMO 设自动失效倒计时，到期界面锁定；提供"续期体验 / 转为正式合作"引导按钮。
- **FR-4.4.5** 续期：平台与企业客户约定 DEMO 失效期，可再次续期。

### 4.5 联盟链 + 法币自动分账 — P0（合规校正版）
> ⚠️ **合规红线（最高优先级，据市场调研 5.10）**：中国实行"币链分离"（银发〔2026〕42 号），**不得使用公链代币 / 稳定币**。分账必须采用 **联盟链 + 人民币法币 + 实名 + 持牌支付/银行托管**。

- **FR-4.5.1** 分账规则写入**联盟链智能合约**（如腾讯云 TBaaS / 蚂蚁链），满足"资金流、信息流、发票流"三流合一。
- **FR-4.5.2** 按里程碑触发：**2-of-3 多签**（专家 / 企业 / 平台三方签章），纠纷期由平台仲裁介入（据链盟/智汇联独立印证）。
- **FR-4.5.3** 资金流经持牌支付机构 / 银行托管账户，法币结算，不得触碰虚拟币。
- **FR-4.5.4** 分阶段落地：早期**先不用区块链**，采用传统支付托管分账；待月均交易达 **100 万+** 再引入联盟链存证 + 法币自动分账（据智汇联分阶段建议）。
- **FR-4.5.5** 费率：平台 take rate 参考 **10–20%**（对标 Catalant/一品威客），链上清算费锚定 **1–3%**（对标文旅分账）；对外统一表述为"**透明智能分账协议**"，不提代币（据 DeepSeek 版包装警示）。

### 4.6 积分与推荐裂变 — P0
- **FR-4.6.1** 积分规则（据项目规范）：新成员 **+10 分**；推荐一个成员 **+2 分**；**推荐成员不重复，重复不计入**。
- **FR-4.6.2** 推荐关系上链记录去中心化图谱，确保非重复成员（对标 Braintrust Connector）。
- **FR-4.6.3** 防刷：微信活体检测 / 实名，或 DID（Gitcoin Passport）链上信用评分，低分账户不发积分（据链盟报告）。
- **FR-4.6.4** 积分须与平台权益**深度绑定**（降低抽佣、优先派单、抵扣年 License），方能持续裂变（据 DeepSeek 版核心发现）。

### 4.7 平台数据看板（实时统计） — P0
按项目规范，平台方实时统计以下类型（**超过一年后同时提供各环比指标**）：

| 统计类型 | 说明 | 优先级 |
|----------|------|--------|
| 专家人数 | 入驻专家总数 | P0 |
| 企业人数 | 入驻企业代表总数 | P0 |
| AI 专家人数 | 入驻 AI 技术人才总数 | P0 |
| 产品个数 | 已发布方案/技能包总数 | P0 |
| DEMO 产品个数 | 进行中/历史 DEMO 总数 | P0 |
| 日交易量 | 当日分账交易额 | P0 |
| 周交易量 | 当周分账交易额 | P0 |
| 月交易量 | 当月分账交易额 | P0 |
| 年交易量 | 当年分账交易额 | P0 |
| 环比指标 | 运营满一年后提供（日/周/月/年环比） | P1 |

### 4.8 收费与授权模式 — P0
最终产品按与客户达成的协议，支持三种类型（据项目规范）：
- **FR-4.8.1** **按年收费 LICENSE**：限期有效，到期需续费；
- **FR-4.8.2** **永久授权**：一次性买断；
- **FR-4.8.3** **到某日期前自动失效**：设定失效日，到期自动失效（与 DEMO 失效机制同源）。

## 5. 基准垂直行业 MVP 场景（制造业）

> 选择**制造业**为标杆垂直行业依据：知微行易以工业 AI 跑通 2 亿收入、30 个 500 强客户，强印证"专家经验 + AI 交付"在制造业可行；市场调研结论点 6 明确"优先制造业等垂直行业验证"；链盟 STEP2 亦锁定传统制造 35+ 专家。

### 5.1 DEMO 场景：小型制造工厂「生产排产与交期风险预警」
- **企业痛点**：订单波动大、排产靠老师傅经验、交期经常延误、缺乏数据化预警。
- **三方角色演示**：
  1. **35+ 生产管理专家**定义排产约束（设备产能、工序先后、换型时间、优先级规则、交期红线）；
  2. **AI 人才**据规则配置一个轻量"排产与交期预警 Agent"，输入脱敏订单数据，输出排产草案 + 高风险订单预警（演示数据）；
  3. **企业代表**在游戏化界面中点选/拖拽审阅排产草案、打回/批准，并看到"延误天数模拟"前后对比（演示数据）。
- **核心功能演示（<50%/核心<20%）**：NDA 在线签署、专家+AI 人才联合工作台极小部分（输入规则→输出草案）、基于合约的分账模拟（企业确认一阶段即触发"智能合约"给专家/AI 人才钱包加款，演示数据）、三方实时消息与修改留痕。
- **数据要求**：全部虚构且显著标注"演示"；自动失效倒计时；提供"续期/转正式"按钮。
- **替代快赢场景**（可选）：连锁餐饮「大众点评差评处理」（据 DeepSeek 版示例）——专家定话术标准 → AI 人才配 Agent 输出差评回复草稿 → 企业游戏化审阅。

## 6. 非功能需求

- **合规（最高优先级）**：分账层严格遵循 §4.5 联盟链 + 法币要求；专家网络行业因凯盛融英事件合规收紧，须建多轮审核 + 合规留痕；税务、涉外数据传输提前法务介入。
- **安全**：NDA 数字水印 + 核心逻辑留云端 + 到期撤 Key（§4.3）；实名/活体防刷（§4.6）。
- **多端发布（降维，据链盟报告）**：**首期拒绝多套原生开发**，采用 **Uni-app / Flutter 跨端渲染**，先极速输出**微信小程序 + 移动端自适应 H5** 闭环验证，再打包 App / PC 壳（省约 80% 成本）。后续再按规范扩展网页版（Win/Harmony/Linux）、独立 App（安卓/iOS/HarmonyOS）、PC 原生、多渠道应用市场。
- **性能**：95 分位响应 < 200ms（对标 API 测试规范）；DEMO 交互实时反馈。
- **多语言**：先仅**简体中文**；其他语言版本根据具体情况后续推进。

## 7. 阶段划分（据 DeepSeek 版后续规划）

| 阶段 | 目标 | 关键交付 |
|------|------|----------|
| **DEMO 阶段** | 验证三方合作与付费意愿 | 本 PRD 的 §4.1–§4.8 最小闭环 + §5 制造业 DEMO |

| **全功能阶段** | 规模化与多行业 | 多垂直行业技能包市场、多语言、全平台原生、自建应用市场 |

## 9. 风险与合规

| 风险 | 等级 | 缓解 |
|------|------|------|
| 区块链合规（币链分离） | 🔴 最高 | §4.5 联盟链+法币+多签；对外称"透明智能分账协议" |
| 35+ 专家学习畏难 | 🟠 高 | 专家端极傻瓜化；AI 辅助生成名片/方案 |
| 三方冷启动 | 🟠 高 | 单点突破→双边扩张→三方闭环；先制造业垂直验证；内部"模拟专家"保底 DEMO 体验 |
| 多端资金链 | 🟠 高 | 跨端框架先攻小程序+H5，后 App/PC 壳 |
| 积分裂变防刷 | 🟡 中 | 实名/活体/DID 信用评分 |
| 撮合初期成功率低 | 🟡 中 | 平台投入内部 AI 人才/模拟专家保证 DEMO 体验 |

## 10. 验收标准与 KPI

- **功能验收**：§4.1–§4.8 全部 P0 功能可用；DEMO 功能开放度实测 ≤50%、核心 ≤20%；演示数据显著标注；到期自动失效+续期可用。
- **合规验收**：分账层通过法务评审，确认无公链代币/稳定币，采用联盟链+法币+持牌托管。
- **业务 KPI（MVP 期末）**：
  - 种子：30 专家 + 20 AI 人才 + 20–30 家企业完成 DEMO 测试；
  - 付费意愿：≥ X 家企业愿预付小额验证费（目标值待运营设定）；
  - 分账信任：区块链自动分账被打消回款顾虑的比例（问卷）；
  - 看板：专家/企业/AI 人数、产品数、DEMO 数、日/周/月交易量实时可查。
