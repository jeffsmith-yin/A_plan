# 融智桥（RongZhiQiao）

> 把"懂行的老专家"和"懂 AI 的年轻人"粘在一起，用 AI 帮中小微企业真正解决问题。

融智桥是一个面向**中小微企业**的四方协作平台，连接四类角色：**行业专家 · AI 人才 · 企业客户 · 平台方**。平台用 AI 做粘合剂，让经验与算力彼此需要，并基于区块链 / W3 合约规则实现各方自动结算。

---

## 痛点与我们的回答

| 痛点 | 融智桥的做法 |
|---|---|
| 大龄专家经验丰富却不懂 AI，难以变现 | 提供"AI 辅助推广 + 解决方案交付"的产品形态，专家只需提供专业经验 |
| 企业家对 AI 了解少，不会综合用 AI 解决企业问题 | 平台派 AI 人才在合适阶段自动介入，把 AI 能力落到具体业务 |
| 专家与 AI 人才彼此找不到、协作难 | 统一入驻（名片式）、实时交流进展、反馈问题、按合约自动付账 |
| 怕泄密、又想展示效果 | 先做最小 DEMO（演示数据），参与方签 NDA，DEMO 到期自动失效、可续期 |

---

## 四方角色

- **行业专家**：经验持有者，提供专业判断与解决方案内容
- **AI 人才**：在不同阶段、不同平台自动介入，承接 AI 落地工作
- **企业客户（代表）**：需求方，使用平台获取解决方案、参与 DEMO
- **平台方**：运营与治理，实时统计各类数据，制定与执行合约规则

> 入驻采用"名片"方式：姓名（或昵称）、电话、微信、职务、个人介绍（重点）。

## 当前状态：DEMO v0.7.1

当前仓库是**最小可用 DEMO**，用于展示对用户痛点的部分解决能力：

- ✅ 四方面协作平台的在线交互雏形（含游戏化基础互动）

- ✅ 登录：用户名+密码 与 手机号 双模式

- ⚠️ 数据为**演示数据**，非真实数据；核心功能占比 < 20%
- ⚠️ 区块链 / W3 合约自动结算、推荐积分等按路线图规划中

## 如何本地运行 DEMO

```bash
# 1. 进入前端项目
cd demo-frontend/rongzhiqiao-demo

# 2. 安装依赖
npm install

# 3. 启动开发服务器
npm start
# 浏览器打开 http://localhost:3000

# 4.（可选）生产构建，产物在 build/，由 proxy.js 静态托管
npm run build
```

**体验账号**：超级管理员 `admin / admin`（也可用手机号模式注册首位用户自动成为超级管理员）。
